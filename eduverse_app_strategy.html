One person, multiple hats. One app handles it gracefully.
Same Login → Different World
When a user logs in, Supabase returns their role(s). The app renders the matching dashboard. If they have multiple roles, a role-switcher chip appears in the header — just like your existing designs already show.

Role Switching — Technical Flow
How Supabase + Flutter + Riverpod handle this seamlessly
STEP 01
Login
User logs in with phone/email. Supabase Auth validates credentials.
STEP 02
Role Fetch
Auth token decoded → user_roles table queried. Returns list of assigned roles + school_id.
STEP 03
Riverpod State
currentRoleProvider set. All widgets rebuild based on active role. Navigation tree changes.
STEP 04
Dashboard Render
GoRouter redirects to role-specific home route. Different bottom nav, quick actions, data.
STEP 05
Role Switch Tap
User taps chip in header → updates currentRoleProvider → full UI re-renders instantly. No re-login.
STEP 06
RLS Protection
Every Supabase query filtered by role. Even if UI bug, DB enforces access at row level.
